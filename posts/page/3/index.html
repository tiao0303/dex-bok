LP 分成 = 70%
协议收入 = $15 万/天

技术基础设施

1. 私有节点 ⭐⭐⭐⭐⭐

为什么需要

  • 公开 mempool 会被抢跑
  • 需要快速广播交易
  • 避免被其他搜索者发现

搭建方案

方案一：自建 Geth 节点
成本：$500/月（服务器）
延迟：50-100ms
难度：中等

方案二：租用节点
提供商：Alchemy、Infura、QuickNode
成本：$200-1000/月
延迟：20-50ms
难度：低

方案三：Flashbots Protect
成本：免费
延迟：100-200ms
难度：低

配置示例（Geth）

# 启动命令
geth --http --http.api eth,net,txpool \
     --ws --ws.api eth,net \
     --txpool.pricelimit 1 \
     --miner.etherbase 0xYOUR_ADDRESS

2. Flashbots Bundle ⭐⭐⭐⭐⭐

什么是 Bundle

  • 一组打包的交易
  • 直接发送给验证者
  • 不经过公开 mempool
  • 避免被抢跑

发送流程

1. 构建交易（包含套利逻辑）
2. 签名
3. 打包成 Bundle
4. 发送到 Flashbots Relay
5. 等待区块包含
6. 成功则执行，失败则回滚

代码示例（Python）

from flashbots import Flashbots

# 初始化
flashbots = Flashbots(provider, signer)

# 构建 Bundle
bundle = [
    {
        "from": "0x...",
        "to": "0x...",
        "value": 0,
        "data": "0x...",
        "gasPrice": 1000000000
    }
]

# 发送
result = flashbots.send_bundle(bundle, target_block_number)

3. 智能合约 ⭐⭐⭐⭐

为什么需要合约

  • 原子性执行（多步操作）
  • Gas 优化
  • 逻辑复杂度高

套利合约模板

// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

interface IUniswapV2Router {
    function swapExactTokensForTokens(...) external;
}

contract Arbitrage {
    address public owner;
    
    constructor() {
        owner = msg.sender;
    }
    
    function executeArbitrage(
        address tokenA,
        address tokenB,
        uint256 amount
    ) external onlyOwner {
        // 步骤 1: 在 Uniswap 买入
        IUniswapV2Router(uniswapRouter).swapExactTokensForTokens(...);
        
        // 步骤 2: 在 Sushiswap 卖出
        IUniswapV2Router(sushiRouter).swapExactTokensForTokens(...);
        
        // 步骤 3: 利润转回 owner
        IERC20(tokenA).transfer(owner, profit);
    }
    
    modifier onlyOwner() {
        require(msg.sender == owner, "Not owner");
        _;
    }
}

期权+DEX 组合策略 - 对冲风险增强收益

为什么需要期权对冲

LP 的痛点

问题 影响 期权解决方案
无常损失 币价波动亏损 Put 期权保护
踏空风险 币价上涨少赚 Call 期权参与
收益不稳定 APY 波动大 期权权利金收入

策略一：LP+Put 期权保护 ⭐⭐⭐⭐⭐

原理

持有：ETH + USDC 做 LP
买入：ETH Put 期权（行权价低于当前价）

效果：
- ETH 下跌：Put 期权盈利，对冲 LP 损失
- ETH 上涨：LP 收益增加，仅损失权利金

实战案例

场景：ETH = $1800，做 LP $10,000

操作：

1. $5,000 ETH + $5,000 USDC → Uniswap LP
2. 买入 ETH Put 期权
   - 行权价：$1600
   - 到期：30 天
   - 权利金：2% ($100)

结果对比：

ETH 价格 无对冲 LP 有对冲 LP
$1500 (-17%) $8,500 $9,400 ✅
$1800 (0%) $10,000 $9,900
$2100 (+17%) $11,200 $11,100 ✅

推荐平台

  • Ribbon Finance：自动化对冲

流动性挖矿收益优化 - 让 APY 翻倍技巧

收益构成分析

三大来源

来源 占比 稳定性
交易手续费 30-50% ⭐⭐⭐⭐⭐
流动性奖励 40-60% ⭐⭐⭐
治理代币空投 10-20% ⭐

真实 APY 计算

名义 APY = 50%
奖励代币跌幅 = -30%/年
Gas 成本 = 5%/年
真实 APY = 50% - 30% - 5% = 15%

优化策略一：复投频率 ⭐⭐⭐⭐⭐

复投 vs 单利

频率 APY(名义 50%) Gas 成本 净收益
每日 64.8% 18% 46.8%
每周 56.4% 4% 52.4% ✅
每月 51.2% 1% 50.2%
不复投 50.0% 0% 50.0%

最佳复投公式

最佳频率 = sqrt(年 APY / Gas 成本比)

示例：
APY = 50%
Gas 成本 = $50/次
持仓 = $10,000
→ 每周复投最优

DEX 套利策略详解 - 从零开始赚无风险收益

什么是套利

套利 = 利用价格差异，低买高卖，赚取无风险收益

套利特点

  • ✅ 理论上无风险
  • ✅ 收益即时锁定
  • ⚠️ 需要快速执行
  • ⚠️ Gas 费可能吃掉利润

策略一：三角套利 ⭐⭐⭐⭐

原理

USDC → ETH → WBTC → USDC
     ↑              ↓
     └──── 利润 ←───┘

案例演示

假设价格：

  • USDC/ETH = 1800
  • ETH/WBTC = 0.065
  • WBTC/USDC = 28500

计算：

1. 10000 USDC → 5.55 ETH
2. 5.55 ETH → 0.36075 WBTC
3. 0.36075 WBTC → 10281 USDC
4. 利润：281 USDC (2.81%)

执行要点

  1. 速度第一：用 Flashbots 避免抢跑
  2. Gas 优化：选择低峰时段
  3. 滑点控制：设置合理上限
  4. 监控工具：自建价格监控脚本

风险

  • 价格快速变化
  • 交易失败损失 Gas
  • MEV 机器人竞争

DeFi 税务基础

应税事件

行为 是否应税 说明
买入加密货币 否 法币→crypto
卖出加密货币 是 crypto→法币
Swap 交易 是 crypto→crypto
提供流动性 可能 视司法管辖区
质押奖励 是 视为收入
空投 是 收到时计税

主要国家/地区

美国（IRS）

  • 资本利得税：短期 10-37%，长期 0-20%
  • 收入税：挖矿/质押奖励按收到时市价

中国

  • 加密货币交易：目前政策不明朗
  • 建议：咨询专业税务师

新加坡

  • 长期持有：免税

DAO 治理完全指南 - 参与协议决策

什么是 DAO 治理

DAO（去中心化自治组织）让代币持有者参与协议决策：

  • 费用调整
  • 资金分配
  • 协议升级
  • 合作伙伴关系

治理流程

1. 提案阶段

  • 论坛讨论（Discourse）
  • 温度检查投票
  • 正式提案提交

2. 投票阶段

  • 链上投票（Snapshot/链上合约）
  • 投票权：代币数量 + 锁定时长
  • 法定人数要求（通常 4-10%）

3. 执行阶段

  • Timelock 等待期（通常 2-7 天）
  • 多签执行
  • 链上生效

主流治理平台

平台 特点 代表项目
Snapshot 链下签名，无 Gas UNI、AAVE
Tally 链上投票 MKR、COMP
Boardroom 多链支持 ENS、LDO

治理策略

委托投票

  • 没时间研究？委托给活跃代表
  • 选择与你有共同利益的 delegate

治理挖矿

  • 参与投票获得额外奖励
  • 注意奖励代币的解锁和价格

提案技巧

  • 先在论坛获得支持
  • 明确执行细节
  • 考虑实施成本
