攻击一：重入攻击（Reentrancy）

原理

攻击流程：
1. 攻击者合约调用目标合约
2. 目标合约转账给攻击者
3. 攻击者 fallback 函数再次调用目标
4. 此时目标合约状态未更新
5. 重复提款，直到 Gas 耗尽

经典案例：The DAO (2016)

时间：2016.06
损失：$6000 万（当时 ETH 价格）
影响：导致以太坊分叉（ETH/ETC）

漏洞代码：
function withdraw(uint amount) {
    require balances[msg.sender] >= amount;
    msg.sender.call.value(amount)();  // 外部调用
    balances[msg.sender] -= amount;   // 状态更新在后
}

问题：
- 外部调用在状态更新之前
- 攻击者可在 balances 更新前再次调用

攻击合约示例

contract Attacker {
    Target target;
    
    function attack() public payable {
        target.deposit.value(msg.value)();
        target.withdraw(msg.value);
    }
    
    function() payable external {
        if (address(target).balance >= msg.value) {
            target.withdraw(msg.value);  // 重入
        }
    }
}

定义

pragma cvl 0.1;

import "./IERC20.sol";

contract ERC20Spec {
    IERC20 token;
    
    // 不变量：总供应量恒定
    invariant totalSupplyInvariant() {
        token.totalSupply() == INITIAL_SUPPLY;
    }
    
    // 不变量：余额之和等于总供应量
    invariant balanceSumInvariant() {
        uint256 sum = 0;
        for (address user : all_users) {
            sum += token.balanceOf(user);
        }
        sum == token.totalSupply();
    }
    
    // 规则：转账后余额正确更新
    rule transferUpdatesBalances(address from, address to, uint256 amount) {
        require token.balanceOf(from) >= amount;
        require to != address(0);
        
        uint256 balanceFromBefore = token.balanceOf(from);
        uint256 balanceToBefore = token.balanceOf(to);
        
        token.transfer(msg.sender, to, amount);
        
        assert token.balanceOf(from) == balanceFromBefore - amount;
        assert token.balanceOf(to) == balanceToBefore + amount;
    }
    
    // 规则：授权不能被滥用
    rule allowanceNotAbused(address owner, address spender, uint256 amount) {
        require token.allowance(owner, spender) == 0;
        require msg.sender != spender;
        
        token.transferFrom(msg.sender, owner, amount);
        
        assert token.allowance(owner, spender) == 0;
    }
}

AMM 核心公式推导

恒定乘积公式（Uniswap V2）

基本公式

x * y = k

其中：
x = Token A 数量
y = Token B 数量
k = 常数

价格推导

边际价格 = dy/dx

对 x * y = k 求导：
y + x * dy/dx = 0
dy/dx = -y/x

所以：价格 P = y/x

滑点公式

买入 Δx 的 Token A：
新价格 P' = (y - Δy) / (x + Δx)

滑点 = (P' - P) / P
     = Δx / (x + Δx)

结论：
- 池子越小，滑点越大
- 交易越大，滑点越大

恒定和公式（Curve StableSwap）

基本公式

x + y = k

价格恒定为 1:1
适合稳定币交易

StableSwap 混合公式

A * n^n * Σx_i + D = A * D * n^n + D^(n+1) / (n^n * Πx_i)

其中：
A = 放大系数
n = 资产数量
D = 不变量
x_i = 各资产数量

A → ∞ 时：趋近恒定和
A → 0 时：趋近恒定积

放大系数 A 的作用

A 值 曲线形状 适用场景
1 标准恒定积 波动资产
100 中间态 弱相关资产
10000 接近恒定和 稳定币

集中流动性公式（Uniswap V3）

核心创新

只在价格区间 [P_a, P_b] 内提供流动性

公式：
L = √(x * y) （在价格范围内）

实际流动性 = L * (√P_b - √P_a)

资本效率提升

效率提升倍数 = P_b / (P_b - P_a)

示例：
当前价格 P = $1800
区间 [1600, 2000]
效率提升 = 2000 / (2000-1600) = 5 倍

区间 [1700, 1900]
效率提升 = 1900 / (1900-1700) = 9.5 倍

虚拟储备推导

在区间 [P_a, P_b] 内：
x_real = L * (1/√P - 1/√P_b)
y_real = L * (√P - √P_a)

当 P 超出区间：
- P < P_a：全部为 x（Token A）
- P > P_b：全部为 y（Token B）

无常损失数学分析

无常损失公式推导

基础推导

初始状态：
- 价格 P0
- 存入 x0 个 Token A，y0 个 Token B
- 总价值 V0 = x0 * P0 + y0

价格变化后：
- 新价格 P1 = r * P0（r 为价格变化倍数）
- 池子自动调整：x1 * y1 = k

LP 价值：
V_lp = x1 * P1 + y1

持有价值：
V_hold = x0 * P1 + y0

无常损失公式

IL = V_lp / V_hold - 1

简化后：
IL = 2√r / (1+r) - 1

其中 r = P1/P0（价格变化倍数）

- 通过门槛：>50% 或>67%
- 反对门槛：可触发二次投票

2. 经济防范 ⭐⭐⭐⭐

投票权设计

方案：
- 锁仓时间越长，投票权越大
- 防止短期操纵

公式：
投票权 = 代币数量 × 锁仓系数

锁仓系数：
1 天 = 1x
30 天 = 2x
90 天 = 3x
365 天 = 4x

反贿赂机制

方案：
- 投票保密（直到结束）
- 增加贿赂难度
- 披露大额投票

质押要求

提案门槛：
- 质押一定数量代币
- 提案失败则没收
- 防止垃圾提案

3. 社区防范 ⭐⭐⭐⭐

多签治理

关键决策：
- 需要多签钱包执行
- 5/9 或 7/11 签名
- 分散权力

社区监督

机制：
- 论坛充分讨论
- 第三方审计提案
- 媒体监督

紧急暂停

权限：
- 社区可投票暂停治理
- 发现攻击时紧急响应
- 多签也可触发

4. 避免权力滥用
