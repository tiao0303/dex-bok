无常损失详解 - LP 最大的隐形成本

什么是无常损失

无常损失（Impermanent Loss, IL）= 提供流动性 vs 持有 的价值差异

简单说：币价波动时，LP 可能比直接持有亏钱。

经典案例

场景：ETH = $1800 时存入 LP

操作 存入 ETH 涨到$3600 后价值
持有 1 ETH + 1800 USDC $5400
LP 1 ETH + 1800 USDC $4242
差异 - -$1158 (-21.5%)

这就是无常损失。

计算公式

IL = (2 * sqrt(price_ratio) / (1 + price_ratio)) - 1

price_ratio = 新价格 / 原价格

MEV 防护指南 - 避免被机器人收割

什么是 MEV

MEV（Maximal Extractable Value）= 矿工/验证者可提取价值

简单说：机器人监控公开交易池，通过特殊手段从你的交易中获利。

常见 MEV 攻击

1. 抢跑（Front-running）

手法：

  1. 机器人发现你的大额买单
  2. 出更高 Gas 抢先买入
  3. 价格被推高后，卖给你获利

损失：滑点增加 0.5-5%

2. 三明治攻击（Sandwich）

手法：

你的交易：买入 10 ETH 的 Token A

机器人操作：
1. 抢先买入 Token A（推高价格）
2. 你的买单执行（价格更高）
3. 立即卖出 Token A（获利）

损失：可达 10%+

3. 清算抢跑

手法：

  1. 监控借贷协议清算线
  2. 发现可清算仓位
  3. 抢先执行清算获利

影响：借款人损失清算罚金

防护工具

1. Flashbots Protect ⭐⭐⭐⭐⭐

原理：私有 RPC，交易不公开到 mempool

使用：

RPC 地址：
https://rpc.flashbots.net

MetaMask 设置：
1. 添加新网络
2. 输入上述 RPC
3. 切换到 Flashbots 网络交易

效果：免疫 99% MEV 攻击

Curve StableSwap 解析 - 稳定币交换的王者

为什么需要 Curve

传统 AMM（如 Uniswap）对稳定币效率极低：

问题：USDC 和 DAI 价格应该始终≈1:1，但 Uniswap 的 x * y = k 公式会导致不必要的滑点。

例子：

  • 用 Uniswap swap 100 万 USDC → DAI
  • 滑点可能达到 0.5%+
  • 损失 $5,000+

Curve 的 StableSwap 公式专为稳定币设计，滑点可降低 10-50 倍。

StableSwap 公式原理

核心思想

结合两种公式的优点：

公式 特点 适用场景
恒定和 x + y = k 滑点极低 价格完全稳定
恒定积 x * y = k 保证流动性 价格波动大

StableSwap 通过放大系数 A 动态调整：

  • 价格接近 1:1 → 接近恒定和（低滑点）
  • 价格偏离 → 接近恒定积（保护流动性）

公式示意

StableSwap = A * (恒定和) + (恒定积)

A 越大 → 越接近恒定和 → 滑点越低
A 越小 → 越接近恒定积 → 流动性越安全

流动性管理策略 - 从被动 LP 到主动管理

三种 LP 策略对比

策略 难度 收益潜力 风险 适合人群
被动 LP ⭐ ⭐⭐ ⭐⭐ 新手
主动调仓 ⭐⭐⭐ ⭐⭐⭐ ⭐⭐⭐ 进阶
集中流动性 ⭐⭐⭐⭐⭐ ⭐⭐⭐⭐⭐ ⭐⭐⭐⭐⭐ 专业

策略一：被动 LP

操作方式

  1. 存入 50:50 双币
  2. 获得 LP Token
  3. 坐收手续费，不做任何操作

优点

  • 简单，一次设置即可
  • 无需关注市场波动
  • 适合长期看好双币

缺点

  • 资金效率低
  • 无常损失完全暴露
  • APY 通常较低（5-15%）

推荐场景

  • 稳定币对（USDC/DAI）
  • 强相关资产（ETH/stETH）
  • 长期持有信念强

策略二：主动调仓

核心思路

定期 rebalance 回 50:50 比例，降低无常损失。

操作步骤

1. 监控池子比例
2. 当偏离>20% 时触发
3. 卖出多的，买入少的
4. 重新存入池子

优点

  • 降低无常损失 30-50%
  • 可在震荡市获利
  • 灵活性高

缺点

  • 需要频繁操作
  • 产生额外 gas 费
  • 可能错过单边行情

工具推荐

  • DeFi Saver：自动 rebalance
  • Gelato：条件触发执行
  • 自建脚本监控

AMM 原理详解 - 自动做市商核心机制

什么是 AMM

自动做市商（Automated Market Maker）是去中心化交易所（DEX）的核心机制，用数学公式代替传统订单簿来实现资产交易。

核心公式：恒定乘积

x * y = k
  • x = Token A 的数量
  • y = Token B 的数量
  • k = 常数（流动性总量保持不变）

公式含义

无论交易如何进行，池中两种 token 的乘积始终保持不变。这导致：

  • 买入 Token A → A 减少，B 增加 → A 价格上涨
  • 卖出 Token A → A 增加，B 减少 → A 价格下跌

AMM 工作流程

graph LR
    A[用户存入双币] --> B[成为 LP]
    B --> C[获得 LP Token]
    C --> D[交易产生手续费]
    D --> E[LP 获得收益]
  1. 提供流动性：用户存入等值的双币（如 1 ETH + 1800 USDC）
  2. 获得凭证：收到 LP Token，代表池子份额
  3. 交易发生：其他用户 swap，支付手续费（通常 0.3%）
  4. 收益分配：手续费累积到池中，LP 按比例分享

关键概念

术语 说明 重要性
流动性池 存放交易对的智能合约 ⭐⭐⭐⭐⭐
LP Token 流动性凭证，可随时赎回 ⭐⭐⭐⭐⭐
滑点 大额交易导致的价格偏移 ⭐⭐⭐⭐
无常损失 币价波动导致的相对损失 ⭐⭐⭐⭐⭐

实战要点

小额交易（<1000 USD）

  • 滑点可忽略不计
  • 直接用 Uniswap/Sushiswap

大额交易（>10000 USD）

  • 分多笔执行
  • 使用聚合器（1inch、Matcha）
  • 考虑使用 Flashbots 避免 MEV

选择池子

  • 看深度 > 看 APY
  • 深度不足时，高 APY 可能是陷阱
  • 检查历史 TVL 稳定性
