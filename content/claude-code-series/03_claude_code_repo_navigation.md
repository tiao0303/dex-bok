# Claude Code 大仓库导航：让 AI 深度理解你的代码库

## 面对大仓库的挑战

当你接手一个数万行的代码库时，Claude Code 是最好的导航工具。

## 文件理解

### 快速了解项目结构
```
claude
帮我了解这个项目的整体结构，重点关注：
1. 入口文件在哪里
2. 主要模块有哪些
3. 配置文件在哪里
```

### 深度理解单个文件
```
读取 @src/core/Exchange.sol 并解释：
- 这个合约的核心数据存储结构
- 主要的外部调用接口
- 安全相关的设计
```

## 代码搜索

### Grep 搜索
```
帮我搜索所有调用了 Uniswap V3 Quoter 的地方，
输出每个调用的文件位置、行号和上下文
```

### 语义搜索
```
找出所有"授权转账"相关的代码（approve + transferFrom），
标注出可能的安全风险点
```

## 跨文件追踪

### 追踪数据流
```
追踪一笔 USDT 转账的完整路径：
从用户点击 Swap 开始，到合约最终执行
经过哪些文件？哪些函数？
```

### 理解依赖关系
```
帮我画出 @src/core/ 目录下各合约的依赖关系图，
用文字描述即可
```

## 安全审查

### 常用审查模式
```
对这个目录 @contracts/security/ 做全面安全审查：
1. 重入攻击风险
2. 整数溢出/下溢
3. 授权绕过
4. 逻辑错误

对每个问题，说明：文件位置、严重程度、修复建议
```

### 特定漏洞检查
```
检查所有 external/public 函数是否有：
- 非零地址检查
- 授权检查
- 余额边界检查
- reentrancy guard
```

## 性能分析

### Gas 优化
```
分析 @contracts/SwapAggregator.sol 的 gas 消耗：
1. 找出最贵的 5 个操作
2. 提出具体优化方案
3. 估算每个优化能节省多少 gas
```

## 实战案例

### DeFi 聚合器代码审查

```
项目路径：~/defi-aggregator

任务：
1. 理解架构：用户如何发起一笔 Swap，路径是怎样的
2. 找出风险点：哪些地方可能被 MEV 机器人攻击
3. 给出安全清单：上线前必须修复的问题

输出格式：
## 架构理解
[文字描述]

## 风险点
| 位置 | 风险 | 严重度 | 建议 |
|------|------|--------|------|

## 上线前检查清单
- [ ] ...
```

## 命令速查

| 需求 | 命令 |
|------|------|
| 了解结构 | `claude "分析项目结构"` |
| 搜索代码 | `/grep "pattern" **/*.sol` |
| 读文件 | `cat @file` 或直接引用 |
| 查依赖 | `claude "追踪 X 的调用链"` |
| 安全审查 | `/security-review @path` |
| Gas 分析 | `/gas-profile @file` |
