## AMM 核心公式推导

### 恒定乘积公式（Uniswap V2）

#### 基本公式
```
x * y = k

其中：
x = Token A 数量
y = Token B 数量
k = 常数
```

#### 价格推导
```
边际价格 = dy/dx

对 x * y = k 求导：
y + x * dy/dx = 0
dy/dx = -y/x

所以：价格 P = y/x
```

#### 滑点公式
```
买入 Δx 的 Token A：
新价格 P' = (y - Δy) / (x + Δx)

滑点 = (P' - P) / P
     = Δx / (x + Δx)

结论：
- 池子越小，滑点越大
- 交易越大，滑点越大
```

---

### 恒定和公式（Curve StableSwap）

#### 基本公式
```
x + y = k

价格恒定为 1:1
适合稳定币交易
```

#### StableSwap 混合公式
```
A * n^n * Σx_i + D = A * D * n^n + D^(n+1) / (n^n * Πx_i)

其中：
A = 放大系数
n = 资产数量
D = 不变量
x_i = 各资产数量

A → ∞ 时：趋近恒定和
A → 0 时：趋近恒定积
```

#### 放大系数 A 的作用
| A 值 | 曲线形状 | 适用场景 |
|------|----------|----------|
| 1 | 标准恒定积 | 波动资产 |
| 100 | 中间态 | 弱相关资产 |
| 10000 | 接近恒定和 | 稳定币 |

### 集中流动性公式（Uniswap V3）

#### 核心创新
```
只在价格区间 [P_a, P_b] 内提供流动性

公式：
L = √(x * y) （在价格范围内）

实际流动性 = L * (√P_b - √P_a)
```

#### 资本效率提升
```
效率提升倍数 = P_b / (P_b - P_a)

示例：
当前价格 P = $1800
区间 [1600, 2000]
效率提升 = 2000 / (2000-1600) = 5 倍

区间 [1700, 1900]
效率提升 = 1900 / (1900-1700) = 9.5 倍
```

#### 虚拟储备推导
```
在区间 [P_a, P_b] 内：
x_real = L * (1/√P - 1/√P_b)
y_real = L * (√P - √P_a)

当 P 超出区间：
- P < P_a：全部为 x（Token A）
- P > P_b：全部为 y（Token B）
```

## 无常损失数学分析

### 无常损失公式推导

#### 基础推导
```
初始状态：
- 价格 P0
- 存入 x0 个 Token A，y0 个 Token B
- 总价值 V0 = x0 * P0 + y0

价格变化后：
- 新价格 P1 = r * P0（r 为价格变化倍数）
- 池子自动调整：x1 * y1 = k

LP 价值：
V_lp = x1 * P1 + y1

持有价值：
V_hold = x0 * P1 + y0
```

#### 无常损失公式
```
IL = V_lp / V_hold - 1

简化后：
IL = 2√r / (1+r) - 1

其中 r = P1/P0（价格变化倍数）
```

#### 数值验证
| 价格变化 | r | IL 计算 | IL 结果 |
|----------|-----|---------|---------|
| +10% | 1.1 | 2√1.1/2.1 - 1 | -0.23% |
| +25% | 1.25 | 2√1.25/2.25 - 1 | -1.4% |
| +50% | 1.5 | 2√1.5/2.5 - 1 | -5.7% |
| +100% | 2.0 | 2√2/3 - 1 | -21.5% |
| -50% | 0.5 | 2√0.5/1.5 - 1 | -21.5% |

### V3 无常损失分析

#### 区间内 IL
```
IL_v3 = IL_v2 * (P_b - P_a) / (√P_b - √P_a)²

结论：
- 区间越窄，IL 越大
- 但收益也越高
- 需要权衡
```

#### 被甩出损失
```
当价格超出区间 [P_a, P_b]：

P < P_a：
- 全部变为 Token A
- 损失 = (P0 - P) / P0

P > P_b：
- 全部变为 Token B
- 踏空收益 = (P - P0) / P0
```

## 最优费用率设计

### 费用率对 LP 收益影响

#### 收益公式
```
LP 收益 = 交易量 × 费用率 × LP 份额

年化 APY = (日交易量 × 365 × 费用率) / TVL
```

#### 费用率与交易量关系
```
交易量 = f(费用率)

费用率↑ → 单笔收益↑ → 交易量↓
费用率↓ → 单笔收益↓ → 交易量↑

最优费用率 = argmax(费用率 × 交易量)
```

#### V3 多档位设计
| 档位 | 适合资产 | 理论依据 |
|------|----------|----------|
| 0.01% | 稳定币 | 低波动，高频率 |
| 0.05% | 主流币 | 中等波动 |
| 0.30% | 长尾资产 | 高波动，低频率 |
| 1.00% | Meme 币 | 极高波动 |

## 资本效率优化

### 效率定义
```
资本效率 = 相同交易量所需资金 / 实际所需资金

V2 效率 = 1x
V3 效率 = P_b / (P_b - P_a)
```

### 最优区间选择

#### 基于波动率
```
最优区间宽度 = 2 × 日波动率 × √T

示例：
ETH 日波动率 = 5%
预期持有期 T = 7 天
最优宽度 = 2 × 5% × √7 = 26.5%

区间：[P×0.87, P×1.13]
```

#### 基于历史数据
```
步骤：
1. 统计过去 30 天价格范围
2. 计算 95% 置信区间
3. 设置略宽于该区间的 LP 区间
4. 平衡被甩风险与效率
```

### 动态再平衡策略

#### 再平衡触发条件
```
条件 1：价格接近区间边界
if P < P_a × 1.1 or P > P_b × 0.9:
    rebalance()

条件 2：效率下降过多
if 当前效率 < 初始效率 × 0.5:
    rebalance()

条件 3：固定时间间隔
if days_since_last_rebalance > 7:
    rebalance()
```

#### 再平衡成本
```
成本 = Gas 费 + 滑点损失 + 手续费

收益 = 效率提升带来的额外收益

决策：
if 收益 > 成本：
    执行再平衡
```

## 高级 AMM 设计

### 曲线参数优化

#### 动态 A 值（Curve）
```
A = f(池子不平衡度)

池子平衡时：A 值高（低滑点）
池子不平衡时：A 值低（保护流动性）

公式：
A = A_base × (1 + imbalance_factor)
```

#### 动态费用（Uniswap V3 改进）
```
费用率 = f(波动率，交易量)

高波动时：费用率↑（保护 LP）
低波动时：费用率↓（吸引交易）

实现：
fee = base_fee × (1 + volatility_multiplier)
```

### 预言机集成

#### TWAP 预言机
```
时间加权平均价格（TWAP）

计算：
TWAP = Σ(price_i × time_i) / Σtime_i

用途：
- 防止价格操纵
- 衍生品结算
- 跨链价格参考
```

#### 预言机操纵攻击
```
攻击方式：
1. 闪电贷大额交易
2. 瞬间改变池子价格
3. 利用被操纵价格借贷/清算

防御：
- 使用 TWAP（多区块平均）
- 增加操纵成本
- 多预言机聚合
```

## 数学工具与代码

### Python 实现 IL 计算器
```python
import math

def impermanent_loss(price_ratio):
    """计算无常损失"""
    return 2 * math.sqrt(price_ratio) / (1 + price_ratio) - 1

def v3_capital_efficiency(price_lower, price_upper, price_current):
    """计算 V3 资本效率"""
    if price_lower >= price_upper:
        return 1
    return price_upper / (price_upper - price_lower)

def optimal_fee_rate(volume_elasticity, base_volume):
    """计算最优费用率"""
    # volume = base_volume * (1 - elasticity * fee_rate)
    # revenue = fee_rate * volume
    # 求导得最优解
    return 1 / (2 * volume_elasticity)
```
