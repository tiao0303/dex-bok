# T2 进阶实战 06：AMM 原理详解 — 理解 DEX 的核心引擎

在上一节中，我们了解了 DEX 的基础概念。本节我们来深入理解 DEX 的核心引擎——**AMM（Automated Market Maker，自动做市商）**。理解 AMM 的原理，是所有进阶 DeFi 操作的根基。

---

## 一、AMM 是什么？

### 传统做市商 vs AMM

在传统金融中，**做市商**是提供流动性的专业机构。他们在订单簿上同时挂出买单和卖单，赚取买卖价差（bid-ask spread）。

```plaintext
传统订单簿模式：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
买单队列                    卖单队列
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
用户B 买入 2 ETH @ 1990    用户D 卖出 1 ETH @ 2010
用户A 买入 1 ETH @ 1980    用户C 卖出 3 ETH @ 2020
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
        ↑ 差价 = 2010 - 1990 = 20 USD
```

**AMM 则完全取消了订单簿**，用一个数学公式取而代之。无论你是买入还是卖出，智能合约根据公式自动报价，无需任何对手方等待。

---

## 二、恒定乘积公式：AMM 的灵魂

### 公式本身

AMM 最核心的公式是**恒定乘积（Constant Product）**：

```
x × y = k
```

其中：
- **x** = 流动性池中 Token A 的数量
- **y** = 流动性池中 Token B 的数量
- **k** = 常数（交易前后乘积不变）

### 直观理解

想象一个装满水的两个连通容器：

```plaintext
         容器 A (x)          容器 B (y)
           ┌───┐              ┌───┐
           │   │◄────────────►│   │
           │████│              │████│
           │████│              │████│
           └───┘              └───┘
            水                   水

数学关系：x × y = k（总水量不变）
```

当你从容器 A 向外倒水（买 Token A），容器 B 的水位就会自动上升（Token B 增加）。倒得越多，水位差越大，价格就越高——这就是价格发现的自动机制。

## 三、AMM 交易的实际计算

### 场景：ETH/USDC 池

假设一个流动性池初始状态：

```
ETH 数量 (x) = 100
USDC 数量 (y) = 200,000
k = 100 × 200,000 = 20,000,000
```

**当前价格** = 200,000 ÷ 100 = **2000 USDC/ETH**

### 计算买入 1 ETH 需要多少 USDC

套入公式：

```
x_new × y_new = k
(100 + 1) × y_new = 20,000,000
y_new = 20,000,000 ÷ 101 ≈ 198,019.80 USDC

用户需要支付：200,000 - 198,019.80 = 1,980.20 USDC
实际成交价格：1980.20 USDC/ETH
```

> ⚠️ 注意：实际成交价格 **1980.20** 与市价 **2000** 有轻微偏差，这就是**滑点（Slippage）**。

### 分辨辨：买入数量对价格的影响

| 买入 ETH 数量 | 需支付 USDC | 实际均价/ETH | vs 市价偏差 |
|:---:|:---:|:---:|:---:|
| 0.1 ETH | 198.02 | 1,980.20 | -0.99% |
| 1 ETH | 1,980.20 | 1,980.20 | -0.99% |
| 5 ETH | 10,256.41 | 2,051.28 | +2.56% |
| 10 ETH | 22,222.22 | 2,222.22 | +11.11% |
| 20 ETH | 50,000.00 | 2,500.00 | +25.00% |

这张表说明了一个关键原则：**大额交易滑点高，分批执行优于一次性大单**。

## 四、流动性质押（LP）的经济学

### 成为 LP 的流程

任何人都可以向流动性池存入两种代币（等值），成为**流动性提供者（LP）**，并获得 **LP Token** 作为凭证。

```plaintext
Alice 想成为 LP：

Step 1: 存入 10 ETH（价值 $20,000）
Step 2: 存入 20,000 USDC（价值 $20,000）
Step 3: 获得 LP Token（代表她在池子中的份额）
Step 4: 等待交易手续费分红

池子总量（存入前）：100 ETH + 200,000 USDC
Alice 占比：20,000 / 240,000 ≈ 8.33%
```

### LP Token 的价值逻辑

LP Token 不是凭空印发的，而是根据你在池子中的**份额比例**铸造：

```
LP Token 铸造量 = 总供应量 × (个人存入价值 ÷ 池子总价值)
```

当你赎回时，用 LP Token 换回对应份额的两种代币 + 该期间累积的手续费。

## 五、AMM 的三大关键概念

### 1. 滑点（Slippage）

滑点是**成交价与预期价的偏差**。

- **原因**：池子深度不足，或交易量相对池子太大
- **计算**：滑点 % = (实际成交价 - 预期价) ÷ 预期价 × 100%
- **应对**：设置**滑点容忍度（Slippage Tolerance）**，通常 0.5%-1%

```javascript
// 实际操作中的滑点设置（ethers.js）
const overrides = {
  slippageTolerance: 0.005, // 0.5%
};

// 或在 DEX UI 中直接设置
```

### 2.无常损失（Impermanent Loss，IL）

这是 LP 最需要理解的风险。

**定义**：将代币存入 AMM 池 vs. 简单持有相比，因价格变化导致的潜在损失。

```
案例：初始 ETH = $2,000，存入 1 ETH + 2000 USDC

存入池子后，ETH 涨到 $4,000：

持有方式：1 ETH = $4,000 + 2000 USDC = $6,000
LP 方式：池子自动 rebalance
  → 池子变为 ~0.707 ETH + ~2,828 USDC
  → 总价值 ≈ $5,657

无常损失 = $5,657 - $6,000 = -$343（-5.7%）
```

> 💡 "无常"意味着：若价格回到初始点，损失消失。但实践中大多数 LP 会长期承受这部分损失。

**无常损失计算公式**：

$$IL = 2 \times \frac{\sqrt{price\_ratio}}{1 + price\_ratio} - 1$$

当价格比 = 1（价格不变）时，IL = 0  
当价格比 = 4（涨 4 倍）时，IL ≈ -13.4%

### 3. 手续费与 APY

LP 的收益来自两部分：

```
LP 总收益 = 交易手续费收入 + 流动性挖矿奖励

手续费收入 = 池子总交易量 × 手续费率 × LP 份额占比
```

以 Uniswap V2 为例：

| 手续费率 | 每日池子交易量 | LP 份额 | 日收益 |
|:---:|:---:|:---:|:---:|
| 0.30% | $1,000,000 | 5% | $15 |
| 0.30% | $1,000,000 | 1% | $3 |

年化计算：日收益 $15 × 365 ÷ $20,000（初始存款）≈ **27.4% APY**

## 六、AMM 的类型对比

除了恒定乘积，还有其他 AMM 公式：

| 类型 | 公式 | 适用场景 | 代表协议 |
|------|------|----------|----------|
| **恒定乘积 (CPMM)** | x × y = k | 通用，任意代币对 | Uniswap V2 |
| **恒定和 (CSMM)** | x + y = k | 零滑点，但可能耗尽 | 极少使用 |
| **StableSwap** | 混合 x+y 与 x×y | 稳定币交易 | Curve |
| **恒定几何平均数** | x × y × z = k | 多资产池 | Balancer |
| **动态联合曲线** | 自定义参数 | 实验性 | 各类新协议 |

> 🔑 **记住**：没有完美的 AMM 公式，每种设计都是权衡。稳定币需要低滑点，波动资产需要深度流动性。

## 七、实战操作指南

### 如何计算最优交易金额

使用 Python 可以快速计算滑点：

```python
def calculate_swap_output(x: float, y: float, dx: float, fee: float = 0.003) -> dict:
    """
    计算 AMM 交易输出
    x, y: 池子中两种资产的数量
    dx: 输入数量
    fee: 手续费率（Uniswap V2 默认 0.3%）
    """
    k = x * y
    
    # 扣除手续费后的输入
    dx_after_fee = dx * (1 - fee)
    
    # 交易后的 x
    x_new = x + dx_after_fee
    
    # 新的 y
    y_new = k / x_new
    
    # 输出量
    dy = y - y_new
    
    # 实际成交价
    price = dx / dy
    
    # 滑点（假设市价为 y/x）
    spot_price = y / x
    slippage = (price - spot_price) / spot_price * 100
    
    return {
        "output": dy,
        "avg_price": price,
        "slippage_pct": round(slippage, 4),
        "fee_paid": dx * fee
    }

# 示例
result = calculate_swap_output(100, 200_000, 1)
print(f"输出: {result['output']:.4f} ETH")
print(f"均价: ${result['avg_price']:.2f}")
print(f"滑点: {result['slippage_pct']:.2f}%")
```

### 选择池子的实用原则

```
✓ 选池子优先看的指标：
  1. TVL（总锁仓量）— 越高越安全
  2. 24h 交易量 — 越高手续费收入越多
  3. APY — 综合收益
  4. 代币对 — 避免新上线或不知名代币

✗ 避开：
  - TVL < $100,000 的池子（容易被操纵）
  - 新建池子（价格发现不充分）
  - 只有奖励没有真实交易量的池子
```

## 八、AMM 的局限性

理解 AMM 的不足，才能更好地使用它：

| 局限 | 说明 | 应对方式 |
|------|------|----------|
| **滑点问题** | 大额交易价格冲击大 | 分批成交或用聚合器 |
| **无常损失** | 币价单边波动时 LP 受损 | 稳定币对或对冲策略 |
| **多代币风险** | LP 必须持有两种代币 | 主动 Rebalance |
| **前端运行（MEV）** | 机器人抢跑大单 | 使用私有交易 |
| **价格预言机依赖** | 初始价格来源有风险 | 参考多个数据源 |

## 九、总结

AMM 用一个简洁的 `x × y = k` 公式，彻底重构了资产交易的方式：

1. **无需对手方**：任何人都可以随时买卖
2. **无需订单簿**：数学公式自动定价
3. **无需许可**：智能合约代码即法律
4. **流动性可组合**：LP 份额可在其他协议中再次利用

理解 AMM 的核心机制——恒定乘积、滑点、无常损失——是所有 DeFi 进阶操作的基础。下一节我们将对比 Uniswap V2 和 V3 的具体差异，理解集中流动性的革命性创新。
